# Deepfake Detection: CNN RGB vs Grayscale and Adversarial Robustness

Binary CNN deepfake detector comparing RGB vs grayscale inputs, with adversarial robustness evaluation under PGD (white-box) and transfer (black-box) attacks, and PGD adversarial training (Madry et al., 2018) as defense.

Course project for **Intelligenza Artificiale e Sicurezza** (Università di Cagliari).

---

## Problem

CNN-based deepfake detectors are known to be vulnerable to evasion adversarial attacks in both white-box and black-box scenarios (Masood et al., 2023; Carlini & Farid, 2020; Neekhara et al., 2021). This project:

1. Trains two CNN detectors (RGB and grayscale) on a balanced real/fake image dataset.
2. Evaluates their robustness under PGD (full gradient access) and transfer attacks (no target knowledge).
3. Hardens both models via PGD adversarial training and re-evaluates robustness.
4. Compares the robustness/clean-accuracy trade-off across RGB vs grayscale, white-box vs black-box, original vs defended models.

---

## Dataset

- 600 real + 600 fake images (perfectly balanced, 1200 total).
- Real: **CelebA** and **FFHQ**.
- Fake: generated by **AttGAN** and **StarGAN**.
- Folder prefix encodes the label: `0_*` → real, `1_*` → fake.
- Images resized to 100×100 pixels, normalized to [0, 1], then min-max normalized using training set statistics.

Split: 60% training / 20% validation / 20% test (80/20 train-test, then 75/25 inside training).

---

## Architecture

Both models share the same architecture, differing only in the input channels:

| Block | Layer |
|-------|-------|
| 1 | Conv2D(32, 3×3) → ReLU → MaxPool(2×2) → Dropout(0.25) |
| 2 | Conv2D(64, 3×3) → ReLU → MaxPool(2×2) → Dropout(0.50) |
| Classifier | Flatten → Dense(2) → Softmax |

- **RGB**: input (100, 100, 3), 87,106 trainable params
- **Grayscale**: input (100, 100, 1), 86,530 trainable params
- Optimizer: Adam, Loss: MSE, Batch size: 4, Max epochs: 50, Early stopping on `val_loss` (patience=15)

## Methodology

The notebook is structured in 5 phases:

- **Phase 0** — Agent and environment analysis (Italian)
- **Phase 1** — Dataset preparation and analysis
- **Phase 2** — CNN implementation (RGB and grayscale)
- **Phase 3** — Training, validation, testing
- **Phase 4** — Results analysis

Followed by the advanced component on adversarial robustness:

- **Step 1** — White-box PGD attack on original models
- **Step 2** — Black-box transfer attack on original models
- **Step 3** — PGD adversarial training (Madry et al., 2018)
- **Step 4** — PGD and transfer attack on adversarially trained models
- **Step 5** — Final comparison

### Attacks

**PGD (white-box)** — untargeted evasion attack, maximizes classification loss while constraining the perturbation to the L-∞ ball of radius ε:

> max L(x+δ, y; θ),  s.t. ||δ||∞ ≤ ε

40 iterations, step size α=0.01, random start inside the ε-ball, clipped to [0, 1].

**Transfer attack (black-box)** — adversarial examples crafted on a surrogate model and transferred to the target without access to its gradients. Two directions tested:
- RGB surrogate → Grayscale target
- Grayscale surrogate → RGB target

### Defense

**Adversarial training** implements the min-max formulation:

> min_θ E[max_δ L(x+δ, y; θ)]

At each batch, PGD adversarial examples are generated (inner maximization), then weights are updated by minimizing a combined loss (outer minimization):

> L = (1 − α) · L_natural + α · L_adversarial,  with α = 0.5

Validation loss is monitored on adversarial examples. Training ε = 0.01.

## Results

### Clean accuracy

| Model | Accuracy | AUC | Final epoch |
|-------|----------|-----|-------------|
| RGB | 97.92% | 0.997 | 23 |
| Grayscale | 98.33% | 0.996 | 49 |

### Robust accuracy

| ε | RGB Orig. | RGB Adv. | Gray Orig. | Gray Adv. |
|---|-----------|----------|------------|-----------|
| 0.00 | 97.92% | 99.17% | 98.33% | 97.92% |
| 0.01 (PGD) | 62.08% | 87.92% | 71.67% | 85.42% |
| 0.02 (PGD) | 9.58% | 41.25% | 20.83% | 47.08% |
| 0.05 (PGD) | 0.00% | 0.83% | 0.00% | 0.83% |
| 0.01 (Transfer) | 87.08% | 96.67% | 91.67% | 93.33% |
| 0.02 (Transfer) | 64.17% | 86.25% | 80.42% | 90.00% |
| 0.05 (Transfer) | 2.92% | 19.58% | 19.58% | 39.17% |

### Key findings

- Imperceptible perturbations (ε=0.01) are enough to collapse undefended models from ~98% to ~62–72%.
- Adversarial training recovers ~26 percentage points on RGB and ~14 on Grayscale under PGD at ε=0.01.
- Transfer attack degrades more slowly than PGD — the lack of gradient access limits effectiveness, but the threat remains significant.
- Grayscale is more robust to transfer attacks both before and after defense, due to its lower sensitivity to chromatic perturbations.
- The robustness/clean-accuracy trade-off is small: RGB clean accuracy goes from 97.92% to 99.17%, Grayscale from 98.33% to 97.92%.
- At ε=0.05 the defense does not generalize — both models still collapse.

## How to run

The notebook is designed for **Google Colab** with GPU runtime. The dataset is loaded from Google Drive via PyDrive2.

1. Open `Deepfake_Detection_CNN_RGB_vs_Grayscale_and_Adversarial_Robustness.ipynb` in Colab.
2. Set runtime to GPU.
3. Run all cells. Trained models, attack results, and adversarially trained models are cached under `models/` for reproducibility — re-runs skip training if cached files are found.

### Local execution

```bash
pip install tensorflow keras opencv-python numpy matplotlib scikit-learn pydrive2
jupyter notebook Deepfake_Detection_CNN_RGB_vs_Grayscale_and_Adversarial_Robustness.ipynb
```

Random seeds (`random.seed(123)`, `np.random.seed(123)`, `tf.random.set_seed(123)`) and `train_test_split(random_state=42)` are fixed for reproducibility.
